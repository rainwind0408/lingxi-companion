# 灵犀学伴 - 算法与数学模型技术文档

本文档详细描述"灵犀学伴"系统中使用的核心算法与数学模型。

## 1. 系统概述

灵犀学伴是一个面向边缘设备的轻量化多智能体情感协同感知系统，采用三阶段处理流程：

```
原始图像 → 多源感知 → 空间融合 → 时间平滑 → 稳定输出
```

## 2. 多源并行感知阶段

### 2.1 表情智能体 (精细感知)

- **模型架构**: MobileNetV3 轻量化卷积神经网络
- **输入**: 通过 Haar 级联分类器定位的人脸 ROI 区域
- **输出**: "困惑/专注"的细粒度置信度概率分布 P_face

### 2.2 行为智能体 (姿态感知)

- **模型架构**: MediaPipe Pose + LSTM (长短期记忆网络)
- **处理流程**: 视频帧 → 骨骼关键点坐标 → LSTM 时序建模
- **输出**: 行为情感概率分布 P_pose (识别"托腮"、"低头"、"前倾"等)

### 2.3 环境智能体 (效度评估)

- **模型架构**: TinyCNN (自定义极轻量模型)
- **输入**: 全帧画面降采样
- **输出**: 环境可信度因子 E ∈ [0, 1]

## 3. 空间协同决策阶段 (动态加权融合)

### 3.1 数学模型

系统采用**动态加权融合模型**计算瞬时情感置信度：

$$S = \sum_{i=1}^{n} w_i(E) \cdot P_i$$

其中：
- $S$: 系统输出的最终情感综合置信度
- $i$: 感知智能体索引 (i=1: 表情, i=2: 行为)
- $w_i(E)$: 受环境可信度因子 $E$ 影响的动态权重函数
- $P_i$: 各独立智能体推理得到的情感概率分布矩阵

### 3.2 动态权重函数

权重函数设计为环境相关的连续函数：

```
w_face(E) = {
    0.7,  if E > 0.8 (高可信)
    0.5,  if 0.4 < E ≤ 0.8 (中等)
    0.3,  if E ≤ 0.4 (低可信)
}

w_pose(E) = 1 - w_face(E)
```

权重变化为连续过程而非开关切换，设置过渡区域避免临界状态波动。

### 3.3 三级协商规则

| 级别 | 条件 | 策略 |
|------|------|------|
| 一级 | 高可信一致 (E>0.8, 表情与行为判断一致) | 直接输出共识结果 |
| 二级 | 环境驱动降权 | 动态调整权重 (如 0.7:0.3 → 0.3:0.7) |
| 三级 | 低置信 (最高概率 < 0.4) | 标记"不确定"，避免误判 |

## 4. 时间维度平滑阶段

### 4.1 "3中2"时序滑动窗口

为过滤眨眼、转头等瞬时干扰，系统采用时序平滑：

```
输入: 连续三帧结果 [r1, r2, r3]
输出: 
  - 如果 ≥2 帧结果相同 → 输出稳定状态
  - 否则 → 继续等待下一帧
```

### 4.2 内存复杂度

O(1) 级常数内存开销，仅需存储窗口内三帧状态。

## 5. 仿生机制

系统模拟生物的"感官补偿"机制：

> 当某一感官（视觉表情）受阻时，大脑会自动增强其他感官（行为姿态）的信号增益。

在算法中体现为：当环境质量下降导致表情识别置信度降低时，系统自动降低表情权重，提高对光照不敏感的行为分析权重。

## 6. 性能指标

| 指标 | 目标值 |
|------|-------|
| 端到端延迟 | < 200ms |
| 模型大小 | < 50MB (边缘部署) |
| 准确率 | > 85% (理想环境) |

## 7. 数学符号汇总

| 符号 | 含义 |
|------|------|
| $S$ | 综合情感置信度 |
| $E$ | 环境可信度因子 |
| $w_i(E)$ | 动态权重函数 |
| $P_i$ | 第i个智能体的概率分布 |
| $P_{face}$ | 表情识别概率分布 |
| $P_{pose}$ | 姿态识别概率分布 |
